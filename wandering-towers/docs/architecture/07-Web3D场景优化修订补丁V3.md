## 1. 问题诊断与根因

### 1.1 巫师头旁残留圆形纸片

**现象**：巫师头部已是球体，但头旁边还有一块圆形纸片没去掉。

**根因**：`Wizard3D.tsx` 渲染了**两个**头部圆片：
- 第一个（`translateZ` 朝前的球体头）——保留
- 第二个（`rotateY(90deg)` 侧立圆盘）——多余，从侧角度可见为圆形纸片

**修法**：删除第二个头部 div。

---

### 1.2 乌鸦堡形状不对

**现象**：乌鸦堡颜色没问题，但样式是一块带城垛的平面纸片，不像塔。

**需求**：乌鸦堡可移动到有标记的塔上方，因此造型应**类似普通塔**（柱体+锥顶），才能视觉上「站在塔顶」。

**根因**：`Castle3D.tsx` 用单个 div + 复杂 clip-path 画了平面城堡剪影，没有立体塔身。

**修法**：重写为「八边形柱体 + 锥顶 + 乌鸦纹章」，颜色保留深蓝灰系。

---

### 1.3 地图块衔接错位

**现象**：相邻 tile 的边对不齐。

**根因（两点）**：

1. `Space.tsx` 让 tile 在 **6 边形 / 8 边形间交替**：
   ```ts
   const tileSideCount = Math.abs(data.spaceIndex % 2) === 0 ? 6 : 8;
   ```
   6 边形与 8 边形边宽不同、朝向不同，邻接必然错位。

2. **几何不可能**：16 个 tile 排在 16 环上，相邻 tile 圆心夹角 22.5°（π/8）。八边形边法线间隔 45°（π/4），**无法**同时让两条边对准两个邻居（邻居在 22.5° 处，正好卡在八边形边的中间）。

**修法**：tile 改用 **16 边形**。16 边形边法线间隔 22.5°，正好与 16 环邻居对齐。

**关键计算**（已在代码注释中保留）：

- 邻居圆心夹角：`2π/16 = π/8`（22.5°）
- tile i 朝 i+1 邻居的边法线方向：`θ_i + 9π/16`
- 16 边形（顶点 startAngle=0）边法线：`π/16 + k·π/8`
- 套用现有 `rotateZ(θ_i + π/2)` 后，边法线 = `θ_i + 9π/16 + k·π/8`，k=0 即对准 i+1，k=7 对准 i-1 ✅
- 因此 **16 边形 + 现有旋转角** 即可边边对齐，无需改旋转
- 贴边半径：`ρ = R·tan(π/16) ≈ 342·0.1989 ≈ 68`（R=tileRingRadius）

**同步**：顶面 clip-path 与侧面墙用同一组 `polygonPoints` 生成，杜绝顶面/侧面边错位。

---

### 1.4 塔点击区只剩顶面（最关键）

**现象**：塔只能点中朝上的面，叠塔时无法点中下层塔，只能移动最顶层塔。

**根因**：`Tower3D.tsx` 中只有顶部 `<button>`（朝上八边形面）绑了 `onClick`，8 个**侧面墙体未绑 onClick**。叠塔时：
- 下层塔的顶面被上层塔塔身遮住 → 点不到
- 下层塔的侧面墙没有 onClick → 点了没用
- 结果：只有最顶层塔的顶面可点

**关键观察**：侧面墙**不会被上层塔遮挡**（上层塔身在上层塔的 Z 区间，与下层侧面墙不重叠），且 `backfaceVisibility: hidden` 使背面墙不拦截事件。因此给侧面墙绑 onClick 即可让任意层塔可点，且任意旋转角度都有效。

**修法**：8 个侧面墙 div 绑定同一 `onClick` + `data-interactive` + `cursor: pointer`。

## 4. 风险与回退

- **16 边形 tile**：比八边形更接近圆形，若美术更想要棱角感可回退为八边形（但需接受邻边无法完美对齐）。
- **侧面点击**：依赖浏览对 3D 变换元素的命中测试（Chrome/Safari 均支持）；如遇个别浏览器不命中，可加一层透明 click shield。
- **tile 半径 68**：由数学贴边计算得出，若目视觉有缝/重叠，微调 `tileRadius`。

## 5. 乌鸦纹章标记与塔尺寸修订（2026-06-30 追加）

V3 落地后用户反馈：乌鸦堡对应的纹章标记「只标在地图左上角，塔上没有标记」，且标记位置在地图外。复核规则与实现后，做以下修订。

### 5.1 规则口径复核（哪些位置有乌鸦纹章）

依据 [V2 规则](../../《巫师飞塔 : The Wandering Towers》完整规则与说明文档（V2）.md) §2.1 / §4.3 / §12.2 与 `STANDARD_MAP`（`packages/shared/src/map-definitions/standard.ts`）：

- **地面纹章位**：仅 `space 0`（乌鸦城堡初始位，`groundHasRavenShield`）。
- **带纹章塔**：5 座，**交错分布** `T01/T03/T05/T07/T09`（`hasRavenShield`），普通塔 `T02/T04/T06/T08`。开局 9 塔顺时针位于 `space 1~9`。
- **纹章是印在塔上的固定属性**：塔移动后纹章跟随该塔，不因移动而消失。
- **城堡可落点** = 顶塔带纹章 ∨ 地面带纹章（引擎 `isRavenShieldPosition`）。

即纹章标记应出现在：`space 0` 地面 + 当前作为顶塔的 `T01/T03/T05/T07/T09`。被压在下层的纹章塔，其纹章被上层塔遮挡，不应可见——这与「顶塔带纹章才算合法落点」一致。

### 5.2 旧实现的问题

| 问题 | 根因 |
|---|---|
| 地面标记看着「在地图外」 | `Space.tsx` 把标记画在 `left:4, top:4`，落在 148×148 方格的极角；而 tile 顶面是 16 边形（内切半径 ≈ 66），方格角隙在 16 边形之外 → 视觉上飘在棋盘背景上。 |
| 塔上没有标记 | 同一个 `isRavenShieldPosition` 圆形标记也画在有塔的格子上，但 `translateZ(tileSurfaceZ+3)` 在塔体之下，被塔身遮挡不可见；而 `Tower3D.tsx` 自带的纹章是另一种盾牌形状画在塔顶正中，未与地面标记统一，不易识别。 |
| 塔偏小 | `towerBaseWidth = 62`，相对 148 格偏小。 |

### 5.3 修法

1. **地面纹章**（`Space.tsx`）：
   - 显示条件改为 `isRavenShieldGround && towerLayers.length === 0`——仅在「地面纹章位且无塔遮挡」时显示；一旦该格被塔压住，地面纹章被塔体盖住，交由塔顶纹章接管。
   - 位置从 `left:4, top:4` 改为 `left:26, top:26`，落在 16 边形顶面左上角内（内切半径 ≈ 66，距中心 ≈ 55，完全在面内），不再出界。
   - 样式不变：蓝色径向圆 + 乌鸦剪影三角形。

2. **塔顶纹章**（`Tower3D.tsx`）：
   - 由「正中盾牌」改为与地面同款的「蓝色圆形 + 乌鸦剪影」，放在塔顶面**右上角**（`top:14, right:14`，16px，落在八边形顶面内）。
   - `translateZ(towerLayerHeight + 6)`：本层是顶塔时可见；上层塔一叠上来，本标记即落入上层塔体 Z 区间被遮挡 → 不可见，正好对应「顶塔带纹章才算合法落点」。

3. **塔尺寸**（`visual3d.ts`）：`towerBaseWidth 62 → 80`、`towerTopWidth 58 → 76`。塔八边形半径 40，tile 16 边形内切半径 ≈ 66，仍留约 26px 宽的 tile 顶面边环可见，不完全遮挡地图；邻塔间距 ≈ 133，塔宽 80 不与邻塔重叠。
