# 技术架构与跨平台方案（Mac / Windows / 可选 Docker）

> 版本：1.0　|　创建：2026-06-25　|　关联：[00-开发任务总表](../00-开发任务总表.md)
>
> 解决 V4 要求：开发内容须支持 Mac 与 Windows 均可运行；Docker 在有必要性时使用。

## 1. 技术选型

| 维度 | 选型 | 理由 |
|---|---|---|
| 语言 | TypeScript（strict） | 与 V3 数据字典 JSON Schema、V2 枚举天然契合；前后端共享类型 |
| Monorepo | pnpm workspaces | 对齐 V4 §17 推荐目录；磁盘高效 |
| 运行时 | Node.js 20 LTS | 长期支持，Mac/Windows 官方安装包 |
| 包管理 | pnpm 9 | 跨平台一致；避免 npm 在 Windows 上的嵌套地狱 |
| 前端 | React 18 + Vite 5 | 棋盘 SPA，HMR 快 |
| 引擎 | 纯 TS（无 Node 专有依赖在核心层） | 便于未来编译 WASM / 跑浏览器 |
| 测试 | Vitest | 与 Vite 同构，快 |
| 校验 | Zod | GameState/Event 运行时校验 |
| 随机 | 可种子化 PRNG（如 mulberry32） | 回放可复现；骰子/洗牌服务端权威 |

## 2. 目录结构（落地 V4 §17）

```
wandering-towers/
  package.json                 # 根，含跨平台脚本
  pnpm-workspace.yaml
  tsconfig.base.json
  docker-compose.yml           # 可选部署
  Dockerfile                   # 可选部署
  docs/                        # 文档（V2/V3/V4 + 任务/日志）
  packages/
    shared/                    # 静态定义层 + 类型 + 枚举（前后端共享）
      enums/ types/ constants/ map-definitions/ spell-definitions/ movement-card-definitions/
    engine/                    # 规则引擎（纯逻辑，无 IO）

    server/                    # 服务端权威层（Phase 5 实装；本轮仅骨架+本地适配）
      api/ ws/ persistence/ adapters/
    client/                    # 前端
      ui/ board/ hand/ spell-panel/ log-panel/ debug-panel/
```

## 3. 分层与数据流

```
Client (React) ──ActionCommand──▶ LocalGameAdapter ──▶ RuleEngine.execute()
                                                            │
                              events ◀── ProduceEvents ◀────┤
                                │                           │
                                ▼                           ▼
                          applyEvent() ◀── 重算 GameState ◀── RunRules
                                │
                                ▼
                     GameState + stateVersion ──快照──▶ Client
```

- **规则引擎纯函数化**：`execute(state, command, rng) → { events, endTurn, endgameTriggered }`，不持有可变全局状态，便于回放与并发。
- **LocalGameAdapter**：在本地单机模式下，前端直接调用引擎；接口形态（execute/getState/getEvents/replay）与 V3 后端协议一致，Phase 5 替换为 HTTP/WS RemoteAdapter 时前端零改动。
- **事件流为唯一状态变更来源**：所有变化经 applyEvent，回放天然可用（V3 §21）。

## 4. 跨平台策略（Mac / Windows 原生）

### 4.1 必须遵守
- **脚本跨平台**：`package.json` scripts 不用 shell 专有语法；需要环境变量时用 `cross-env`。
- **路径处理**：代码内一律 `node:path` + `path.join`，禁止字符串拼 `/`。
- **换行符**：`.gitattributes` 强制 `* text=auto eol=lf`，避免 CRLF/LF 混乱。
- **文件名**：全小写 kebab-case，避免 Windows 大小写不敏感冲突。
- **无原生依赖**：引擎与 shared 不引入 node-gyp 类原生模块；若必须，走 optionalDependencies + prebuilt。
- **Node 版本固定**：`.nvmrc` 写 `20`；README 给 Mac(nvm/brew)/Windows(nvm-windows/官方安装包) 安装说明。

### 4.2 开发命令（Mac/Windows 通用）
```
pnpm install
pnpm dev            # 并行启动 client(Vite) + 本地引擎 watch
pnpm -r typecheck
pnpm test           # Vitest 全量
pnpm test:engine    # 仅引擎
pnpm replay:check   # 100 局随机不变量
```

### 4.3 Docker 的定位（可选，非强制）
**结论：本地开发不依赖 Docker；提供 Docker 作为「统一部署/联调/CI」的可选方式。**

引入 Docker 的必要性场景：
1. Phase 5 联机版部署到服务器时，环境一致。
2. CI 环境（GitHub Actions）用 Docker 跑测试，避免 Mac/Windows runner 差异。
3. 想一键起「前端 + 后端 + 持久化」整套。

**不引入**原生依赖，故本地开发无需 Docker 也能跑——这是「Node 原生跨平台 + 可选 Docker」的核心。

## 5. Docker 方案（可选）

### 5.1 Dockerfile（多阶段，单镜像跑 client+server 静态）
```dockerfile
FROM node:20-bookworm-slim AS deps
WORKDIR /app
RUN corepack enable
COPY pnpm-workspace.yaml package.json ./
COPY packages/*/package.json ./packages/*/
RUN pnpm install --frozen-lockfile

FROM deps AS build
COPY . .
RUN pnpm -r typecheck && pnpm -r build

FROM node:20-bookworm-slim AS runner
WORKDIR /app
ENV NODE_ENV=production
COPY --from=build /app /app
EXPOSE 3000
CMD ["node", "packages/server/dist/main.js"]
```

### 5.2 docker-compose.yml（可选三服务）
```yaml
services:
  app:
    build: .
    ports: ["3000:3000"]
    environment:
      - NODE_ENV=production
  # 联机/持久化阶段再加 redis/db
```

### 5.3 跨平台 Docker 注意
- Docker Desktop 在 Mac(Apple Silicon 用 linux/amd64 或 arm64)与 Windows(WSL2)均可。
- 镜像基于 `node:20-bookworm-slim`（Linux），与宿主无关。
- 提供 `pnpm docker:build` / `pnpm docker:up` 脚本，Windows 无需 PowerShell 专有写法。

## 6. 随机性与回放可复现
- 引擎所有随机（洗牌、掷骰）经注入的 `Rng` 接口（mulberry32 + seed）。
- 事件 payload 携带实际随机结果（如骰子点数），回放时不再重新随机，保证 TC-REPLAY-001。
- 服务端权威掷骰（V3 §23.2 方案 A）。

## 7. 开发环境快速就绪（README 节选）
- **Mac**：`brew install node@20 pnpm` 或 `nvm install 20 && corepack enable`
- **Windows**：安装 Node 20 官方包 → `npm i -g pnpm`；建议启用 WSL2 但非必须
- 两平台后续命令完全一致：`pnpm install && pnpm dev`

## 8. 决策记录
| 决策 | 选择 | 替代方案 | 理由 |
|---|---|---|---|
| 语言 | TS | Python/Rust | 文档推荐目录与 TS 类型契合，跨平台最佳 |
| Docker | 可选 | 强制/不用 | 本地开发体验优先，联机部署再用 |
| 前端调用 | LocalGameAdapter 同接口 | 直接调引擎 | 为 Phase 5 联机零改动作预留 |
